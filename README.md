# Image to Hex Gateway

Este projeto é um gateway backend em Node.js (NestJS) que se comunica via gRPC com dois microserviços escritos em C#:

- **ColorAnalyzer**: Recebe uma imagem e retorna as cores dominantes (unário)
- **ImageFilterService**: Recebe uma imagem via streaming bidirecional, aplica um filtro e retorna a imagem processada em chunks

## Requisitos

- Node.js >= 18
- pnpm ou npm
- Serviços gRPC backend rodando (endereços configuráveis)

## Estrutura

- `src/`: Código-fonte do gateway
- `protos/`: Arquivos .proto para comunicação gRPC

## Uso das funções gRPC

O gateway expõe funções para consumir os serviços gRPC:

- `analyzeColors(imageBuffer: Buffer): Promise<string[]>` — Envia uma imagem e retorna as cores dominantes
- `applyFilterStream(imageId: string, filterType: number, imageBuffer: Buffer, chunkSize?: number): Promise<Buffer>` — Envia uma imagem via streaming, aplica o filtro e retorna a imagem processada

### Exemplo de teste

Coloque uma imagem chamada `test-image.jpg` na raiz do projeto e execute:

```sh
pnpm ts-node src/example-grpc-test.ts
```

O script irá:

- Analisar as cores dominantes da imagem
- Aplicar um filtro (filterType=1) e salvar o resultado em `test-image-filtered.jpg`

### Configuração dos endpoints gRPC

Configure os endereços dos serviços backend via variáveis de ambiente:

- `COLOR_ANALYZER_ADDRESS` (padrão: localhost:5281)
- `IMAGE_FILTER_ADDRESS` (padrão: localhost:5281)

Exemplo:

```sh
COLOR_ANALYZER_ADDRESS=localhost:5281 IMAGE_FILTER_ADDRESS=localhost:50052 pnpm ts-node src/example-grpc-test.ts
```
